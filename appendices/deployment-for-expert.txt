Приложение. Инструкция развёртывания для эксперта (подробно, с командами для копирования)

Назначение файла: пошаговый запуск решения на машине проверяющего с командами для терминала, пояснениями и вариантами без Docker. Формат plain text для вставки в документ.

Требования к среде

(список):
Git для клонирования репозитория
Docker Engine и плагин Docker Compose v2 (команда docker compose)
для варианта с крупной модели VOSK — несколько гигабайт свободного места на диске и достаточно оперативной памяти на время загрузки модели в процесс API
свободные порты хоста: 5432, 8000, 8080 (или измените проброс портов в docker-compose.yml перед запуском)

Клонирование репозитория

В каталоге, где удобно хранить проект:

git clone https://github.com/arprspb/y2026f.git
cd y2026f

Дальнейшие команды ниже выполняются из каталога y2026f, если не указано иное.

Вариант 1. Запуск через Docker Compose (рекомендуется для проверки)

Шаг 1. Подготовить каталог модели VOSK

Крупная русская модель по умолчанию в проекте: vosk-model-ru-0.42 (архив порядка 1,8 ГБ). Скачивание и распаковка (пример для Linux или macOS с wget; при отсутствии wget используйте браузер и перенесите архив в каталог models):

mkdir -p models
cd models
wget https://alphacephei.com/vosk/models/vosk-model-ru-0.42.zip
unzip vosk-model-ru-0.42.zip
cd ..

Проверка структуры: должен существовать файл с путём

models/vosk-model-ru-0.42/am/final.mdl

Если после распаковки каталог am оказался на уровень глубже (вложенная папка внутри архива), перенесите содержимое так, чтобы папка am лежала непосредственно внутри models/vosk-model-ru-0.42. Код API при старте ищет маркер am/final.mdl и при необходимости заходит во вложенный подкаталог с тем же именем, но проще сразу выровнять путь вручную.

Шаг 2. Поднять сервисы

Из корня репозитория:

docker compose up --build

Дождаться в логах контейнера api успешного выполнения alembic upgrade head и строки о запуске Uvicorn. Контейнер db должен пройти healthcheck.

Шаг 3. Проверить доступность API

В отдельном терминале:

curl -s http://127.0.0.1:8000/health

Ожидаемое тело ответа в JSON содержит status со значением ok. Реализация эндпоинта:

backend/app/main.py [https://github.com/arprspb/y2026f/blob/main/backend/app/main.py]

Шаг 4. Открыть интерфейс и документацию

(список):
веб-интерфейс: http://localhost:8080
интерактивная документация API (Swagger): http://localhost:8000/docs

Шаг 5. Вход в систему

Учётная запись первого администратора задаётся переменными окружения сервиса api в файле docker-compose.yml: BOOTSTRAP_ADMIN_USERNAME и BOOTSTRAP_ADMIN_PASSWORD. В текущей конфигурации репозитория по умолчанию указаны логин admin и пароль admin. Для проверки на своей машине это удобно; для демонстрации в сети смените пароль и перезапустите compose.

Переменные окружения и файл docker-compose

Для сценария docker compose основные параметры заданы прямо в блоке environment сервиса api в файле:

docker-compose.yml [https://github.com/arprspb/y2026f/blob/main/docker-compose.yml]

Отдельный файл .env в корне для этого сценария не обязателен: строка подключения к PostgreSQL указывает на сервис db по имени хоста db. Том models монтируется в контейнер только для чтения по пути /models/vosk-model-ru-0.42.

Если нужно сменить SECRET_KEY для подписи JWT или пароль администратора, отредактируйте docker-compose.yml в соответствующих строках и выполните перезапуск:

docker compose down
docker compose up --build

Облегчённая модель VOSK (слабое железо)

Скачайте архив small-ru, распакуйте в каталог, например models/vosk-model-small-ru-0.22, убедитесь в наличии am/final.mdl. В docker-compose.yml измените том монтирования и значение VOSK_MODEL_PATH в environment api на согласованный путь внутри контейнера, затем перезапустите compose. Ссылка на small-модель есть в README.

Файл-пример переменных для локального запуска без Docker (другие значения DATABASE_URL и путей):

.env.example [https://github.com/arprspb/y2026f/blob/main/.env.example]

Вариант 2. Локальная разработка (без Docker)

Требуются установленные Python с зависимостями бэкенда, Node.js для фронтенда, PostgreSQL на localhost, ffmpeg в PATH.

Бэкенд: создать виртуальное окружение в каталоге backend, установить пакет в режиме разработки, скопировать .env из примера и выставить DATABASE_URL на вашу базу, VOSK_MODEL_PATH на локальный каталог модели, UPLOAD_DIR. Выполнить миграции и запуск uvicorn по инструкции в README.

Кратко команды из README:

cp .env.example .env

далее правка .env под вашу машину, затем из каталога backend после установки зависимостей:

alembic upgrade head
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

Фронтенд в отдельном терминале:

cd frontend
npm ci
npm run dev

Dev-сервер фронтенда по умолчанию на порту 5173; в CORS бэкенда должны быть перечислены origin с 5173. Полный текст:

README.md [https://github.com/arprspb/y2026f/blob/main/README.md]

Автотесты бэкенда

Из каталога backend с активированным окружением и установленными зависимостями:

pytest tests/ -v

Частые проблемы

(список):
ошибка при старте API про отсутствие модели — проверить путь и наличие am/final.mdl, см. шаг 1
порт занят — остановить другой процесс или изменить сопоставление ports в docker-compose.yml
база не поднимается — дождаться healthcheck db в логах compose перед готовностью api
