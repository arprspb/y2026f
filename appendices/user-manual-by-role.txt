Приложение. Инструкция пользователя по ролям

Назначение: как пользоваться веб-интерфейсом после входа в зависимости от роли учётной записи. Самостоятельная регистрация в системе отключена; логин и пароль выдаёт администратор (первый администратор создаётся при развёртывании сервера по переменным окружения, см. README и docker-compose).

Общие сведения

Адрес интерфейса после запуска по инструкции развёртывания: http://localhost:8080 (или адрес сервера, который выдал администратор).

Вход: на странице входа введите логин и пароль, нажмите кнопку входа. При успехе откроется основной экран приложения.

Выход: в правой части верхней панели нажмите «Выход» — сессия завершится, потребуется снова ввести логин и пароль.

Верхнее меню после входа зависит от роли:

(список):
пункт «Запись» показывается только тем, кому разрешено создавать новые голосовые записи (администратор и оператор записи)
пункт «История» доступен всем авторизованным пользователям
пункт «Пользователи» виден только администратору
рядом с кнопкой выхода отображается ваш логин

Первый экран после входа: для оператора проверки по умолчанию открывается «История»; для администратора и оператора записи — экран «Запись».

Роль: оператор записи

Назначение: диктовать команды в микрофон, просматривать распознавание и сохранять записи в историю.

Экран «Запись» (пункт меню «Запись»):

(нумерованный список):
Нажмите «Начать запись» и разрешите браузеру доступ к микрофону, если браузер запросит разрешение.
Произнесите команду чётко. Во время записи отображается индикатор уровня сигнала.
Нажмите «Стоп» — запись отправляется на сервер, выполняется распознавание речи. Дождитесь окончания надписи о распознавании.
В блоке «Результат распознавания» проверьте текст, извлечённую команду и параметр (идентификатор).
Нажмите «Сохранить в историю», чтобы зафиксировать запись в системе, либо «Перезаписать», чтобы отменить предпросмотр и начать новую запись с шага 1.

После сохранения откроется карточка этой записи. В карточке оператор записи может прослушать аудио и видеть текст; править текст и нажимать «Подтвердить» может только пользователь с правом проверки (см. ниже).

Экран «История»:

(список):
для роли оператор записи в таблице показываются только записи, созданные под вашим логином
для ролей администратор и оператор проверки отображаются все записи всех операторов
поля фильтра: фрагмент команды, параметр, оператор (логин автора записи), интервал дат; кнопка «Фильтровать» обновляет таблицу
щелчок по номеру записи в колонке ID открывает карточку записи

Роль: оператор проверки

Назначение: просматривать историю голосовых команд, при необходимости исправлять распознанный текст по прослушиванию оригинала и подтверждать результат.

Пункт «Запись» в меню у этой роли не отображается — создание новых записей микрофоном недоступно.

Экран «История»: как у оператора записи — подбор записей фильтрами, переход в карточку по ссылке на ID.

Карточка записи (путь вида История, затем выбор строки):

(нумерованный список):
Прослушайте аудио встроенным плеером.
При неверном распознавании отредактируйте текст в поле «Текст (правка)».
Нажмите «Сохранить правку», чтобы записать исправление без завершения проверки.
Когда результат верен, нажмите «Подтвердить» — в системе сохранится факт подтверждения и ваш логин как подтвердившего. Повторное подтверждение не требуется, кнопка подтверждения для уже подтверждённой записи недоступна.

Если поле правки недоступно для редактирования, у учётной записи нет роли проверки; обратитесь к администратору.

Роль: администратор

Назначение: полный доступ к функциям оператора записи и оператора проверки, а также управление учётными записями других пользователей.

В меню доступны «Запись», «История» и «Пользователи».

Работа на экранах «Запись», «История» и в карточке записи — как описано для оператора записи и оператора проверки: администратор может создавать записи, фильтровать историю, править текст и подтверждать записи.

Экран «Пользователи» (только администратор):

(нумерованный список):
В блоке «Новый пользователь» укажите логин, пароль и роль: «Оператор (запись)» или «Оператор (проверка)». Нажмите «Создать». Новый администратор через этот интерфейс не создаётся — учётные записи администратора задаются при развёртывании API (переменные окружения, см. сообщение на странице и README).
В таблице пользователей для каждой строки с ролью оператора можно сменить роль выпадающим списком (запись или проверка). Для пользователя с ролью администратора роль в интерфейсе не меняется.
Кнопка «Блокировать» / «Разблокировать» переключает возможность входа под этой учётной записью. Заблокированный пользователь не сможет войти, пока администратор снова не разблокирует запись.

Рекомендуется выдавать операторам индивидуальные логины и не использовать общий пароль на длительный срок.
