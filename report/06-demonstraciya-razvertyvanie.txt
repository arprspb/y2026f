Инструкция пользователя и видеодемонстрация

По условию задания в состав решения входит развёрнутая инструкция пользователя с учётом ролей. Кратко: вход по логину и паролю (выдаёт администратор); оператор записи работает с экраном «Запись», предпросмотром и сохранением в историю; оператор проверки пользуется «Историей» и карточкой записи для правки текста и подтверждения; администратор дополнительно открывает «Пользователи» для создания учётных записей с ролью оператора записи или проверки, смены роли и блокировки. Пункт «Запись» в меню не показывается пользователям только с ролью проверки.

Инструкция для развёртывания у эксперта

Ниже — минимальная последовательность действий в терминале после клонирования репозитория на машину с установленными Docker и Docker Compose. Команды можно копировать по строкам. Пояснения, смена секретов, локальный запуск без Docker, облегчённая модель VOSK и типичные неполадки вынесены в приложение.

Предусловия: свободны порты 5432, 8000 и 8080 на хосте (или заранее измените проброс в docker-compose.yml).

Клонирование (один раз):

git clone https://github.com/arprspb/y2026f.git
cd y2026f



Скачивание и распаковка модели VOSK (из корня репозитория):

mkdir -p models
cd models
wget https://alphacephei.com/vosk/models/vosk-model-ru-0.42.zip
unzip vosk-model-ru-0.42.zip
cd ..

Проверка: должен существовать файл models/vosk-model-ru-0.42/am/final.mdl. Если структура архива дала лишнюю вложенность, перенесите папки так, чтобы am лежала прямо внутри vosk-model-ru-0.42.

Запуск всех сервисов (из корня репозитория):

docker compose up --build

Дождаться в логах успешных миграций Alembic и запуска Uvicorn в контейнере api.

Проверка API во втором терминале:

curl -s http://127.0.0.1:8000/health

В ответе ожидается JSON с полем status равным ok.

Открыть в браузере:

(список):
интерфейс: http://localhost:8080
Swagger: http://localhost:8000/docs

Вход: логин и пароль начального администратора задаются в docker-compose.yml (BOOTSTRAP_ADMIN_USERNAME, BOOTSTRAP_ADMIN_PASSWORD); в поставляемой конфигурации по умолчанию admin и admin. Перед публичной демонстрацией рекомендуется сменить пароль и секреты, см. приложение.
