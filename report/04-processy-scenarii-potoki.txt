Процессы, сценарии использования и информационные потоки (пункт 2.4 задания)

Соответствие сценариям MVP из задания

Сценарий 1. Оператор распознаёт команду: вход в систему, экран записи, озвучивание, предпросмотр распознавания, сохранение в истории — реализовано экраном записи и цепочкой API preview и confirm.

Сценарий 2. Корректировка результата: оператор проверки открывает карточку из истории, прослушивает аудио, правит текст при необходимости, подтверждает — реализовано экраном детализации и полем подтверждения в базе.

Сценарий 3. Поиск и верификация: список с фильтрами по команде, параметру, дате, оператору — экран истории и запросы к API.

Сценарий 4. Администрирование: создание пользователей с ролью, блокировка — экран «Пользователи» для роли администратор, эндпоинты создания и обновления пользователя.
