Общая цепочка обработки записи голоса

На уровне логики приложения цепочка выглядит так: входной аудиофайл приводится к WAV 16 кГц моно 16 бит; по WAV вызывается VOSK; полученная строка проходит нормализацию для команд; из нормализованного текста извлекаются ключевая фраза и идентификатор; результаты сохраняются в базе и отдаются клиенту. Ниже даны псевдокод и детали по модулям.

Псевдокод: распознавание речи (ASR) по файлу WAV

Вход: путь к файлу в формате PCM 16-bit mono 16 kHz.

Шаги:

(нумерованный список):
Загрузить модель VOSK один раз (singleton), путь к каталогу модели взять из настроек; при первом обращении проверить наличие структуры модели (например am/final.mdl), при необходимости «поднять» вложенную папку из архива (resolve_vosk_model_dir).
Создать KaldiRecognizer с частотой дискретизации из WAV.
В цикле читать фрагменты сэмплов из WAV; для каждого фрагмента вызывать AcceptWaveform; при промежуточном результате с непустым text добавлять фрагмент к списку строк.
После EOF вызвать FinalResult и добавить финальный text при наличии.
Склеить все части через пробел и обрезать краевые пробелы.

Псевдокод: нормализация транскрипта перед извлечением команды

Цель: привести устные числительные к цифрам и склеить последовательности одноразрядных цифр в один токен (чтобы восьмизначный номер не оставался восемью отдельными словами), затем при необходимости склеить блок цифр с «разнесённым» буквенным хвостом, как часто отдаёт ASR.

Функция normalize_transcript_for_commands(raw):

(нумерованный список):
Применить normalize_text: нижний регистр, замена буквы ё на е, схлопывание пробелов.
Разбить строку на токены по пробелам.
Каждый токен, если он совпадает со словарём русских числительных «ноль»…«девять», заменить на одну цифру.
Пройти токены слева направо: если подряд идут односимвольные цифры, объединить их в один токен без пробелов; иначе оставить токен как есть.
Склеить токены пробелами.
Применить итеративную подстановку по шаблону «длинный блок цифр + пробел + хвост из букв/цифр»: если в хвосте есть цифра, склеить цифры и хвост в один токен; если хвост из нескольких однобуквенных слов подряд, тоже склеить с цифровым блоком (код вроде «в б д» после номера); иначе оставить как есть. Повторять до стабилизации.

Псевдокод: извлечение ключевой команды

Функция extract_command(normalized):

(нумерованный список):
Иметь упорядоченный список фраз команд от более длинных к более коротким: отменить регистрацию, отменить обработку, начать обработку, завершить обработку, зарегистрировать.
Для каждой фразы в этом порядке: если фраза входит как подстрока в normalized, вернуть эту фразу.
Иначе вернуть отсутствие команды.

Псевдокод: извлечение идентификаторов

Функция extract_identifiers(normalized, command):

(нумерованный список):
Если command найдена, взять подстроку после первого вхождения фразы команды (хвост); если команда есть и хвост пустой, вернуть пустой список идентификаторов.
Иначе искать по всей строке normalized.
На области поиска применить несколько классов регулярных выражений: смешанные буквенно-цифровые токены; шаблон с лидирующими цифрами и буквенным продолжением; только цифры как отдельные «пробеги»; только буквы длиной от трёх символов с фильтром стоп-слов (номер, плавки, обработку и т.д.).
Собрать пары (позиция, токен), отсортировать по позиции, убрать дубликаты токенов с сохранением порядва первого появления.
Вернуть список строк.

Точность и скорость распознавания речи

Критерий оценивания задания включает точность и скорость на примерах команд из условия. Фиксированные численные показатели в отчёт без проведения измерений на контрольном наборе не вносятся. Рекомендуемая методика для воспроизводимой оценки:

(нумерованный список):
закрепить версию модели VOSK и параметры ffmpeg
подготовить набор тестовых фраз из задания и близких к ним формулировок
для каждой фразы записать аудио в одинаковых условиях микрофона
засечь время от начала запроса до ответа API с финальным транскриптом на том же оборудовании, где будет проверка
сравнить эталонный текст с распознанным по символьной ошибке или по совпадению извлечённой команды и идентификатора

По умолчанию в инструкции к развёртыванию указана крупная русская модель vosk-model-ru-0.42 для повышения качества по сравнению с компактной small-ru; при ограниченных ресурсах можно указать путь к малой модели в переменной окружения. Субъективно при использовании крупной модели на сервере качество распознавания коротких команд с цифрами и буквами выше, чем у малой модели, ценой большего размера на диске и времени загрузки модели в память.
