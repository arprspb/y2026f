Сценарий видеодемонстрации (покрытие сценариев MVP и тест-кейсов задания)

Подготовка к записи

(список):
развёрнуть систему по README (Docker Compose или локально), убедиться что API и веб открываются
подготовить учётные записи: администратор из BOOTSTRAP (например admin или по .env), при необходимости заранее создать двух операторов с разными ролями или показать создание на видео
проверить микрофон в браузере для экрана записи
при желании подготовить заранее список фраз из задания для диктовки

Рекомендуемая последовательность кадров

Блок A. Вход и администрирование (сценарий 4, MVP п.6)

(нумерованный список):
открыть страницу входа, ввести логин и пароль администратора, войти
показать верхнее меню: доступны пункты согласно роли (у админа — Запись, История, Пользователи)
перейти в раздел Пользователи
создать пользователя с ролью оператор записи (operator_record), задать логин и пароль
создать второго пользователя с ролью оператор проверки (operator_verify)
при желании показать смену роли в списке или блокировку учётной записи и обратную разблокировку
выйти из системы (Выход)

Блок B. Оператор записи (сценарий 1, MVP п.1–5)

(нумерованный список):
войти под учётной записью оператора записи
открыть экран Запись
включить запись с микрофона, произнести команду из задания с параметром, например фразу про регистрацию с буквенно-цифровым обозначением или про обработку с числовым идентификатором
остановить запись, дождаться предпросмотра: на экране видны распознанный текст, извлечённая команда и параметр
нажать сохранение подтверждения записи (если в интерфейсе есть шаг подтверждения после предпросмотра)
перейти в История, убедиться что новая запись появилась в списке с временем и оператором

Блок C. Оператор проверки (сценарий 2)

(нумерованный список):
выйти и войти под оператором проверки
открыть История (должны быть видны записи, в том числе чужие)
открыть карточку одной из записей по ссылке с идентификатором
воспроизвести аудио встроенным плеером
изменить текст в поле правки при необходимости, сохранить правку
нажать Подтвердить, убедиться что статус подтверждения и при наличии отображение кто подтвердил отражают изменение
вернуться в список истории

Блок D. Поиск и фильтры (сценарий 3)

(нумерованный список):
на экране История ввести фильтр по фрагменту команды или параметра или дате или имени оператора (один или несколько фильтров)
нажать фильтрацию, показать сокращённый список
открыть найденную запись и кратко показать детали

Блок E. Дополнительно для полноты MVP

(список):
показать страницу Swagger API по адресу из README если нужно продемонстрировать API
кратко показать что публичная регистрация отключена (попытка вызова register возвращает отказ) — опционально через Swagger или текстом за кадром

Итог кадра: кратко проговорить что продемонстрированы сценарии 1–4 и требования к сохранению аудио, распознавания, метаданных и ролей.
